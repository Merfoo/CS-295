Risks and Solutions

There are multiple security issues. One security issue is only having one person be responsible for being super admin, having access to domain name, hosting account, and CMS account. Another security issue is giving the staff ftp access when the don't need it considering they already have accounts with the same access. Another security issue is that the SSL certificate could only be for the domain and not subdomains. Another issue is comments don't have to be processed first would could allow people to post malicious content.

Too Much Responsibility
Split up responsibility among multiple people.
Don't give all the responsibility to one person to begin with.
FTP Access
Revoke FTP access.
Don't give them FTP access to begin with.
SSL Coverage
Get multiple SSL for subdomains or wildcard SSL for domain.
Get multiple SSl for subdomains or wildcard SSL for domain to being with.
Malicious Comments
Remove malcious comments.
Process comments before actually posting them.
Bot Account Detection
Remove all account within a certain time frame to get or based on ip address to remove bot accounts
Add bot detection to the user accounts to prevent bots from being able to add new accounts.
Public Email
Remove email from public view, setup another another email
Set up the contact form such that it tells the server to send the to the public email so that the contact email is never public.
Reliance On One Person
Allow multiple people to be super admins
Don't give only person super admin.
3rd Party Theme Malware
Check with multiple sources if the 3rd party theme can be trusted.
Don't use a 3rd party theme, or use a trusted theme instead.
Revoke Developers Access After Completion
Immediately revoke any permissions the developer has.
After the project is completed, remove any permissions the developer has to the website.
Revoke Access Immediately From Ex-Employees
Immediately revoke any permissions from ex-employee.
Before firing them, revoke any permissions from the ex-employee to prevent damage from them.
