Comparison of Frameworks
  Markup CSS grids CSS Compilers Javascript Libraries Server-side Scripts Data Handling Browser Support Notes
Boilerplate Standard HTML markup provided. CSS grids not included. CSS compilers not included. Comes with jQuery via CDN and local one for fall back and modernizer. Server-side scripts not included. Does not include any tools for external data handling. Browser support is provided via: Browser upgrade not, htaccess, touch/favorite MS touch tiles, normalize stylesheet Simple frameworks meant to get a basic website up and running quickly.
Bootstrap Includes standard HTML markup and many preconfigured components. CSS grids use float properties to mimic pure CSS grid style. Includes precompiled LESS/SASS, can use Grunt to build LESS/SASS. Includes plenty of JS libraries to modify HTML elements behaviors. Server-side scripts not included. Does not include any tools for external data handling. Supports latest version of desktops browsers Chrome, Firefox, Internet Explorer, Opera, and Safari. For mobile the latest versions of Chrome, Firefox, and Safari are supported. More complex framework for and inludes more libraries for a more interactive website.
Angular.js Includes standard HTML markup and allows the creation of reuseable HTML components written in JS. CSS grids not included CSS compilers not included. AngularJS is a JS oriented framework, has tons of modular JS libraries you can use. Server-side scripts not included. Has limited functionality in JS for external data handling. Supports latest versions of Chrome, Firefox, Edge, Internet Explorer, Safari, iOS, Android, and Internet Explorer mobile Framework intended for developers that want to create an interactive website/webapp in typescript or JS.
Ruby on Rails Includes standard HTML markup and has support for template files to make modular components. CSS grids not included. Includes support for SASS through a ruby gem thats installed by default. No JS included by default, supports JS libraries though. Framework is written in and requires Ruby on the server for development. External data handling is one of main pros of Ruby on Rails, has support for connecting to many popular databases. Ruby on Rails is a server side application, browser compatibility depends on the frontend framework beind used. Ruby on Rails intended for developers that want to create scalable applications with their choice of frontend. Has great support and documentation for websites/webapp that use a database.
